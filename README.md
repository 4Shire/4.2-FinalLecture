## Тестирование формы записи на курс "Тестировщик ПО"

Предусловия:
- [X] Выполнен вход на главную страницу сайта https://netology.ru/
- [X] Выбран курс "Тестировщик ПО"
- [X] Нажата кнопка "Записаться"

Валидными данними считать:
* Поле "Имя" - разрешены только русские буквы, дефисы и пробелы, минимум два символа: Иван
* Поле "Телефон" - только цифры (11 цифр), символ "+" спереди: +71234567890
* Поле "Email" - иметь четкую структуру: логин, @, домен: Ivan@mail.ru

#### Позитивное тестирование формы записи на курс "Тестировщик ПО" для зарегистрированных пользователей:
* Авторизироваться  
* Нажать кнопку "Записаться"
* Поля "Имя" и "Телефон" автоматически заполняться данными, указанными при регистрации
* Нажать кнопку "Записаться"

Ожидаемый результат: 

Сообщение - "Заявка принята. Наш менеджер свяжется с Вами в ближайшее время!"

#### Позитивное тестирование формы записи на курс "Тестировщик ПО" для незарегистрированных пользователей:
* Заполнить поле "Имя" валидными данными: Иван
* Заполнить поле "Телефон" валидными данными: +71234567890
* Заполнить поле "E-mail" валидными данными: ivan@mail.ru

## Перечень используемых инструментов:

* IntelliJ IDEA
    - Удобная среда разработки ПО, поддерживает самые распространенные языки программирования и ОС
* GIT
    - Средство облачного храниня и обмена репозиториями
* Selenide
    - Фреймворк для создания автотестов, удобен при тестировании UI
* Java 11
    - Один из наиболее используемых языков програмирования
* Gradle
    - Система управления зависимостями, оптимален при создании проектов с авто-тестами
* JavaFaker
    - Генератор тестовых данных, упрощает заполнение полей ввода при отправке форм
* Lombok
    - Помощь в написании кода автотестов, упрощает процесс создания стандартных методов и конструкторов
* JUnit
    - Платформа для запуска автотестов, наиболее распространенный фреймворк
* Allure
    - Оптимальное средство построения отчетов о проведенном тестировании
    
## Перечень необходимых разрешений/данных/доступов:

* Разрешение на проведение тестирования администрации веб-сайта Нетологии
* Доступ к БД и API для сверки результатов

## Перечень и описание возможных рисков при автоматизации

* Удорожание проекта
* При работе с БД есть риск повреждений самой БД
* Постоянный контроль и актуализация тестов приведёт к увеличению затрат времени
* Нестабильная работа сайта во время тестовых работ
* Доступ к БД может попасть в руки злоумышленников

## Перечень необходимых специалистов для автоматизации:

* Тестировщик-автоматизатор
